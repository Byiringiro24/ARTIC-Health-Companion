# Hospital Manager Dashboard — Complete Role & Feature Breakdown

## Executive Overview

The Hospital Manager serves as the chief administrator for their specific hospital or healthcare facility within the ARTIC Health Companion platform. Unlike the Super Admin who controls the entire system across all hospitals, the Hospital Manager has complete control over their own facility's operations, staff, departments, and clinical workflows. The Hospital Manager is the primary decision-maker for their hospital and acts as the bridge between the Super Admin's global system settings and the day-to-day operations of their healthcare facility. This role is designed to give hospital leadership full autonomy over their operations while maintaining the security and privacy standards required in healthcare.

## Core Philosophy of Hospital Manager Access

The Hospital Manager dashboard is built on the principle of complete facility control with appropriate boundaries around patient privacy and system-wide settings. The Hospital Manager can see everything about their own hospital including staff, patients, departments, financials, and operations but cannot see data from other hospitals or make system-wide changes that affect other facilities. The Hospital Manager has the authority to create staff accounts, manage departments, oversee clinical operations, monitor financial performance, and request new features from the Super Admin when needed. This separation ensures that each hospital operates independently while remaining part of the larger ARTIC ecosystem.

## Sidebar Navigation Structure

The Hospital Manager dashboard features a comprehensive sidebar that organizes all hospital management functions into logical categories. The sidebar is always visible on the left side of the screen, providing quick access to every management function needed to run a healthcare facility. The navigation is divided into primary sections that reflect the Hospital Manager's core responsibilities including hospital overview, staff management, patient management, clinical operations, pharmacy and laboratory, billing and finance, reporting and analytics, communication tools, feature requests, and system settings specific to their hospital. The sidebar also displays the Hospital Manager's identity, hospital name, current subscription tier, and quick access to critical alerts. The design ensures that the Hospital Manager can navigate between any management function in just one or two clicks, making the dashboard highly efficient for daily hospital administration.

## Dashboard Home Tab

The dashboard home serves as the central command center for the hospital showing key performance indicators at a glance. This tab displays real-time statistics including total active patients, today's appointments, staff currently on duty, pending lab results, unread messages, and revenue collected today. The home tab also shows bed occupancy rates, emergency department status, and any critical alerts that require immediate attention such as urgent lab results or patient safety concerns. A visual calendar shows the day's schedule with appointment counts by hour, and a quick overview of pending tasks helps the Hospital Manager stay on top of operational needs. The dashboard also features a quick actions section where the Hospital Manager can perform common tasks like creating a new staff account, scheduling a department meeting, or generating a daily report. This central command center gives the Hospital Manager a complete snapshot of hospital operations at a glance.

## Staff Management Tab

The staff management tab is one of the most critical sections of the Hospital Manager dashboard as it controls all human resources functions for the hospital. This tab displays a complete list of all staff members organized by department with their roles, contact information, current status, and last login date. The Hospital Manager can view detailed staff profiles including employment history, qualifications, certifications, assigned departments, and performance metrics. The tab also includes comprehensive staff management features including the ability to create new staff accounts, edit existing staff information, assign roles and permissions, manage staff schedules, track attendance and leave, and manage performance reviews. The Hospital Manager can also view staff availability, manage shift rotations, and generate staff reports for administrative purposes.

## HR Manager Integration for Staff Creation

The Hospital Manager can delegate staff creation responsibilities to the HR Manager through a specialized permission system. The HR Manager is a sub-role within the hospital that has specific permissions for staff management but limited access to clinical operations. When the Hospital Manager enables HR Manager permissions, the HR Manager gains access to the staff creation and management features while still working under the oversight of the Hospital Manager. This allows the HR Manager to handle the administrative burden of hiring, onboarding, and managing staff records while the Hospital Manager focuses on broader strategic and clinical operations. The HR Manager can create new staff accounts, assign them to departments, set their roles, and manage their employment records. All actions performed by the HR Manager are logged and visible to the Hospital Manager, ensuring complete oversight and accountability.

## Staff Creation and Onboarding Process

When a new staff member needs to be added to the hospital, either the Hospital Manager or the HR Manager initiates the staff creation process through a comprehensive form. The person creating the staff account provides essential information including the new staff member's full name, email address, job role or position, department assignment, qualifications and certifications, start date, employment type whether full-time, part-time, or contract, and any special permissions or access levels needed. The system then automatically generates a secure password for the new staff member and immediately sends a welcome email to their provided email address. This welcome email contains the auto-generated password and provides two distinct options for accessing the system. The first option allows the new staff member to login immediately using the auto-generated password, which enables them to start using the system right away. The second option provides a secure link where the staff member can set their own custom password, which is recommended for better security. The email clearly explains both options and includes security tips about password protection.

## Email Authentication and Password Change Flow

The email authentication system ensures that every new staff member receives their login credentials securely and can choose their preferred authentication method. When the welcome email arrives in the staff member's inbox, they see a professionally designed email from ARTIC Health Companion containing their login credentials. The email clearly displays their email address and the auto-generated password that was automatically created for them. The email then presents two clear options for accessing the system. The first option is to login immediately using the auto-generated password, which is a simple and straightforward approach that allows the staff member to start using the system without any delay. The second option is to click a secure link that takes them to a password setup page where they can create their own custom password. This gives the staff member full control over their security preferences. If the staff member chooses to use the auto-generated password, they can login immediately and later change their password from their profile settings. If they click the link to set their own password, they are redirected to a secure page where they enter and confirm their new password. The system then updates their account and they can login using their custom password going forward. The email also includes security tips about never sharing passwords and contacting support if they did not request this account. The entire flow is designed to be user-friendly while maintaining high security standards, giving staff members the flexibility to choose their preferred authentication method.

## Patient Management Tab

The patient management tab gives the Hospital Manager complete visibility and control over all patient-related operations in the hospital. This tab displays a comprehensive list of all patients with their basic demographic information, registration date, assigned primary care provider, and current status. The Hospital Manager can view patient records, manage patient registrations, oversee patient flow through different departments, and monitor patient satisfaction metrics. The tab includes features for patient registration where staff can add new patients to the system, patient search capabilities to quickly find existing patients, and patient history views to see complete medical records including previous visits, diagnoses, treatments, lab results, and medications. The Hospital Manager can also manage patient referrals, track patient outcomes, and generate patient population reports for administrative and planning purposes.

## Clinical Operations Tab

The clinical operations tab provides the Hospital Manager with oversight of all clinical activities in the hospital. This tab displays real-time information about consultations in progress, waiting room status, emergency department activity, and surgical schedules. The Hospital Manager can monitor doctor schedules, nurse assignments, and clinical workflow efficiency. The tab also includes features for managing clinical protocols, tracking clinical outcomes, monitoring quality indicators, and ensuring compliance with clinical standards. The Hospital Manager can view detailed reports on clinical performance, identify bottlenecks in patient flow, and make data-driven decisions to improve clinical operations. This tab ensures that the Hospital Manager has complete visibility into the clinical side of the hospital while maintaining appropriate boundaries around individual patient clinical data.

## Appointment Management Tab

The appointment management tab gives the Hospital Manager complete control over the hospital's appointment scheduling system. This tab displays a comprehensive calendar view showing all appointments across all departments and providers. The Hospital Manager can view appointments by day, week, or month, see appointment status including scheduled, checked-in, in-progress, completed, or no-show, and monitor appointment volumes and trends. The tab includes features for managing appointment templates, setting provider availability, configuring appointment types, and handling appointment cancellations and rescheduling. The Hospital Manager can also view appointment analytics including no-show rates, appointment volumes by department, and patient wait times. This helps the Hospital Manager optimize scheduling efficiency and improve patient access to care.

## Pharmacy and Medications Tab

The pharmacy and medications tab provides the Hospital Manager with oversight of the hospital's pharmacy operations. This tab displays real-time information about medication inventory levels, pending prescriptions, dispensed medications, and medication safety alerts. The Hospital Manager can monitor drug shortages, track medication usage patterns, manage pharmacy staff schedules, and ensure compliance with medication management protocols. The tab includes features for inventory management where the Hospital Manager can view stock levels, set reorder points, track expiring medications, and manage medication procurement. The Hospital Manager can also view prescription analytics including most prescribed medications, prescription volumes by department, and medication adherence rates for chronic disease patients. This ensures the Hospital Manager has complete visibility into pharmacy operations while maintaining patient privacy around individual prescriptions.

## Laboratory Management Tab

The laboratory management tab gives the Hospital Manager oversight of all laboratory operations in the hospital. This tab displays real-time information about pending test orders, completed tests, critical results, and lab turnaround times. The Hospital Manager can monitor lab workload, track test volumes, manage lab staff schedules, and ensure compliance with lab quality standards. The tab includes features for viewing lab analytics including test volumes by department, average turnaround times, and lab revenue contributions. The Hospital Manager can also track lab inventory levels, manage lab equipment, and monitor lab quality indicators. This ensures the Hospital Manager has complete visibility into laboratory operations while maintaining appropriate boundaries around individual patient results.

## Billing and Finance Tab

The billing and finance tab provides the Hospital Manager with comprehensive oversight of the hospital's financial operations. This tab displays real-time information about revenue collected today, pending invoices, insurance claims status, and payment processing. The Hospital Manager can view detailed financial reports including revenue by department, revenue by service, insurance claim success rates, and patient payment trends. The tab includes features for invoice management where the Hospital Manager can view all invoices, mark payments as received, generate new invoices, and manage write-offs. The Hospital Manager can also manage insurance claims, track claim denials, and monitor claim processing times. The tab also includes budget management features where the Hospital Manager can view department budgets, track expenses, and forecast future revenues. This comprehensive financial oversight ensures the Hospital Manager can make informed business decisions and maintain the hospital's financial health.

## Insurance and Claims Management Tab

The insurance and claims management tab gives the Hospital Manager complete visibility into the hospital's insurance operations. This tab displays all submitted insurance claims, their current status, and any issues that require resolution. The Hospital Manager can view claims by payer, track claim success rates, monitor claim processing times, and manage claim denials and appeals. The tab includes features for insurance eligibility verification where staff can check patient coverage before services are rendered. The Hospital Manager can also view insurance analytics including top payers, claim rejection reasons, and revenue by insurance type. This helps the Hospital Manager optimize insurance operations and improve revenue cycle management. The tab also includes features for managing insurance contracts and tracking contract performance to ensure the hospital is getting optimal reimbursement rates.

## Reporting and Analytics Tab

The reporting and analytics tab provides the Hospital Manager with comprehensive data and insights to make informed decisions. This tab displays all available reports organized by category including clinical reports, operational reports, financial reports, and quality reports. The Hospital Manager can view pre-built reports such as daily census, revenue summaries, department performance, and patient satisfaction scores. The tab includes custom report builder features where the Hospital Manager can create specific reports based on their unique needs, selecting data fields, date ranges, and aggregation methods. The Hospital Manager can schedule recurring reports, export data in various formats, and share reports with other department heads. The analytics dashboard provides visual representations of key metrics including trends over time, comparisons between departments, and performance against targets. This ensures the Hospital Manager has the data needed to make strategic decisions and continuously improve hospital operations.

## Communication Tools Tab

The communication tools tab gives the Hospital Manager complete control over internal and external communications for the hospital. This tab displays the integrated chat system where staff can communicate with each other across all roles including doctors, nurses, pharmacists, and administrative staff. The Hospital Manager can view chat analytics including message volumes, response times, and user engagement. The tab also includes broadcast messaging features where the Hospital Manager can send important announcements to all staff, specific departments, or individual users. The communication tools also include patient communication features where the Hospital Manager can manage patient messaging campaigns, appointment reminders, and health education communications. This ensures that all communications within the hospital are coordinated, professional, and effective.

## Feature Request Tab

The feature request tab allows the Hospital Manager to request access to new features from the Super Admin. This tab displays all features that are currently locked or require approval, organized by category and showing which features are available based on their current subscription tier. The Hospital Manager can browse the full catalog of features, see which ones are not yet enabled, and submit formal requests for access to specific features. When submitting a request, the Hospital Manager provides a justification explaining why the feature is needed and how it will benefit the hospital. The request then goes to the Super Admin for review and decision. The tab also shows the status of all previously submitted requests, whether they were approved or denied, and any conditions attached to approved requests such as usage limits or expiration dates. This ensures a clear and transparent process for accessing new capabilities as the hospital grows and its needs evolve.

## Subscription and Billing Management Tab

The subscription and billing management tab gives the Hospital Manager visibility into the hospital's subscription status and billing history. This tab displays current subscription tier, monthly fee, features included, and renewal date. The Hospital Manager can view the complete invoice history, see upcoming billing dates, and manage payment methods. The tab also shows feature usage statistics indicating how much of the included capacity is being utilized and when the hospital might need to upgrade to a higher tier. The Hospital Manager can view detailed billing breakdowns showing costs by department or service line. This ensures the Hospital Manager has complete financial transparency and can make informed decisions about subscription management and resource allocation.

## System Settings Tab

The system settings tab allows the Hospital Manager to configure all settings specific to their hospital. This tab includes general settings such as the hospital's name, address, contact information, and MOH registration number. Department settings allow the Hospital Manager to create new departments, assign department heads, and configure department-specific workflows. Role and permission settings allow the Hospital Manager to manage which staff members have access to which features and data. Notification settings allow configuration of email and SMS notifications for various events including appointment reminders, prescription alerts, and critical lab results. The settings tab also includes security settings where the Hospital Manager can configure password policies, session timeouts, and access controls specific to their hospital. This ensures the Hospital Manager can customize the system to meet their hospital's unique operational needs.

## What the Hospital Manager Cannot Access

Despite the extensive control the Hospital Manager has over their hospital's operations, there are strict boundaries that must be respected to protect patient privacy and system integrity. The Hospital Manager cannot access patient clinical data including diagnoses, treatment plans, or doctor's notes unless they have a clinical role and are directly involved in patient care. The Hospital Manager cannot see individual patient conversations with healthcare providers as these contain confidential medical information. The Hospital Manager cannot view patient financial details beyond aggregated billing data needed for hospital administration. The Hospital Manager cannot access staff personal information beyond what is needed for employment management such as contact details, role, and department. The Hospital Manager cannot change system-wide settings that affect other hospitals or make changes to global feature configurations. The Hospital Manager cannot view data from other hospitals including their staff, patients, or financial information. The Hospital Manager cannot modify Super Admin-level settings such as pricing, tier configurations, or global feature status. These restrictions are enforced to ensure patient privacy, maintain clear separation between hospitals, and preserve the Super Admin's role as the ultimate system owner and governor.

## Summary of Hospital Manager Responsibilities

The Hospital Manager is the chief administrator of their hospital within the ARTIC Health Companion platform, with complete control over staff management including the ability to delegate staff creation to HR Manager. The Hospital Manager oversees patient operations, clinical workflows, pharmacy and laboratory functions, billing and finances, and all reporting and analytics. The Hospital Manager can request new features from the Super Admin, manage the hospital's subscription, and configure system settings to meet their specific needs. The Hospital Manager cannot access patient clinical data without a clinical role, cannot view data from other hospitals, and cannot make system-wide changes. The entire staff creation process includes auto-generated passwords sent via real email with two options for authentication: immediate login with the auto-generated password or setting a custom password through a secure link. All staff members have the flexibility to choose their preferred authentication method while maintaining security best practices. The Hospital Manager dashboard is designed to give hospital leadership complete operational control while maintaining strict boundaries around patient privacy and system integrity.

Generic note quality indicators without reviewing the actual note content. |
| **Individual Medication Prescriptions** | Prescription records reveal what medications patients are taking, which can disclose sensitive health conditions. The Hospital Manager should never see individual patient prescriptions. This information could be used to discriminate against patients or violate their privacy. | Aggregated medication usage statistics showing which drugs are most prescribed. Pharmacy volume data. Prescription refill rates. Department-level prescribing patterns. Inventory management data that shows stock levels without linking to patients. |
| **Individual Patient Imaging and Reports** | Medical imaging and radiology reports contain identifiable patient information and reveal specific health conditions. Equipment utilization rates. Report turnaround times. Department-level imaging demand data. Equipment maintenance records and performance metrics. |
| **Individual Mental Health Records** | Mental health records are among the most sensitive medical information and receive special protection under law. ### Enforcement Mechanisms

**Database-Level Restriction:**
The database must be configured with row-level security that prevents the Hospital Manager role from querying any patient clinical data. This means that even if the Hospital Manager attempted to directly access the database through a SQL injection or other attack, the database itself would reject the query because the Hospital Manager role lacks the necessary permissions. The patient clinical data tables should have a security policy that specifically excludes the Hospital Manager role from reading any records. The hospital manager role should only be able to query aggregated data views that have been specifically designed for administrative purposes.

**API-Level Restriction:**
The application programming interface must have middleware that intercepts all requests from Hospital Manager accounts and blocks any request that would return individual patient clinical data. This middleware checks every API endpoint that is accessed by a Hospital Manager and verifies that the requested data falls within the approved categories for hospital management. If the request is for patient clinical data, the API immediately returns an access denied error before any data is retrieved from the database. The error message should be generic to avoid revealing what data exists, simply stating that the user does not have permission to access this information.

**UI-Level Restriction:**
The user interface must not display any patient clinical data to Hospital Manager accounts. This means that even if the database or API were compromised and returned clinical data, the user interface would not render it. The UI components that display clinical data should check the user's role before rendering and either show an access denied message or simply hide the clinical data sections entirely. The Hospital Manager dashboard should be designed without any tabs, widgets, or sections that would display individual patient clinical information. Population health dashboards should be carefully designed to show only aggregated, non-identifiable data with clear assurances that individual patients cannot be identified from the information presented.

**Audit and Monitoring:**
All access attempts to patient clinical data by Hospital Manager accounts must be logged and monitored. Any attempt by a Hospital Manager to access clinical data, whether through the API, database, or UI, should be recorded in a security audit log. Attempted access should be detected by the monitoring system and generate an alert for the Super Admin. Repeated or suspicious attempts should automatically trigger security protocols including account suspension and investigation. This ensures that any potential violation of patient privacy is detected and addressed immediately.

**Training and Certification:**
All Hospital Managers must complete annual training on patient privacy and data protection. This training must explicitly cover what data they are not allowed to access and the consequences of violating patient privacy. Hospital Managers should be required to sign a confidentiality agreement confirming they understand these restrictions and will not attempt to access patient clinical data. The training should include real-world examples of privacy violations and their consequences, emphasizing the legal and ethical importance of protecting patient information.

## 2. Other Hospital Data — Complete Restriction

### What Must Be Removed

The Hospital Manager must never see any data from other hospitals, clinics, or healthcare facilities. This is essential for maintaining data privacy between competing or independent healthcare organizations. Even aggregated data about other hospitals should be restricted unless it is specifically anonymized and used for benchmarking purposes with explicit consent.

| Data Type | Why It Must Be Removed | What To Show Instead |
|-----------|----------------------|---------------------|
| **Other Hospital Staff Information** | Staff records from other hospitals are private employment information. The Hospital Manager has no legitimate business need to see staff from other facilities. This could be used for competitive intelligence or recruitment poaching. It violates the privacy of staff members at other hospitals. | Only the Hospital Manager's own hospital staff information. No data about staff at other facilities. Not even aggregated counts of staff at other hospitals. |
| **Other Hospital Patient Data** | Patient records from other hospitals are completely off-limits. The Hospital Manager has no legitimate reason to see patients from other facilities. This would be a massive privacy violation. | Absolutely nothing about patients from other hospitals. This data is entirely restricted and should not be accessible in any form. |
| **Other Hospital Financial Information** | Financial data from other hospitals is competitively sensitive. The Hospital Manager should never see revenues, expenses, or other financial metrics from other facilities. This could be used for competitive advantage. | Only the Hospital Manager's own hospital financial data. No aggregated financial benchmarks from other hospitals unless voluntarily shared and anonymized. |
| **Other Hospital Performance Metrics** | Performance data from other hospitals is confidential and competitively sensitive. The Hospital Manager should not be able to see how other hospitals are performing. This could be used to gain competitive advantage. | Only the Hospital Manager's own hospital performance metrics. Anonymized industry benchmarks that cannot be traced to specific hospitals if available through authorized programs. |
| **Other Hospital Feature Access** | Feature access levels at other hospitals are the business of those hospitals and the Super Admin. The Hospital Manager should not know which features other hospitals have enabled. This could influence competitive strategy unfairly. | Only the Hospital Manager's own hospital feature access. | **Other Hospital Subscription Information** | Subscription tiers and pricing for other hospitals are commercially sensitive. The Hospital Manager should not know what other hospitals are paying. ### Enforcement Mechanisms

**Data Isolation:**
The system must implement complete data isolation where each hospital's data is stored in a separate logical partition. This is typically achieved through multi-tenancy architecture where each hospital has its own database schema or through row-level security that filters all data by hospital ID. Every query against the database must include an implicit filter that only retrieves data for the Hospital Manager's specific hospital. This prevents any accidental or intentional viewing of other hospital data.

**API Scoping:**
All API endpoints that return data to Hospital Manager accounts must be scoped to only return data for their specific hospital. The API should not even have the capability to request data from other hospitals when authenticated as a Hospital Manager. The hospital ID is automatically extracted from the user's authentication token and applied as a filter to all queries. This eliminates the possibility of a Hospital Manager accessing data from other hospitals even through API manipulation.

**UI Data Context:**
The user interface should be explicitly branded with the Hospital Manager's hospital name and should clearly indicate that all data displayed is specific to their facility. This provides visual context and reinforces that the Hospital Manager is operating within their own hospital's data environment. Any attempts to access other hospital data through direct URL manipulation should be blocked by the routing system.

**Audit Monitoring:**
The system must log all requests for data by Hospital Manager accounts and monitor for any patterns that might indicate attempts to access other hospital data. Unusual query patterns, requests for data from other hospitals, or suspicious API calls should generate security alerts. The audit logs should be regularly reviewed by the Super Admin to ensure data isolation is being maintained.

**Audit Monitoring:**
